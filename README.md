# 100 Days of Web Development Learning Plan

Welcome to my **100 Days of Web Development** journey! This repository documents my structured plan to master **Front-End**, **Back-End**, and **Database** technologies in 100 days. Whether you're a beginner or looking to enhance your skills, you can use this repository to follow my learning process and build your own web development skills.

---

## Objective
To learn and apply **Front-End**, **Back-End**, and **Database** skills to build real-world web applications through a 100-day plan. This repository includes my learning roadmap, code, and projects to help you follow along.

---

## How to Use This Repository for Your Learning
To replicate my **100 Days of Web Development** journey using this GitHub repository, follow these steps:

1. **Clone the Repository**:
   - Run `git clone <repository-url>` to download this repository to your local machine.
   - Alternatively, download the ZIP file from the GitHub page and extract it.

2. **Explore the Roadmap**:
   - Read the **Learning Plan** section below to understand the 100-day structure.
   - Each phase (Front-End, Back-End, Databases) is organized by days with topics and projects.

3. **Follow the Folder Structure**:
   - The repository is organized by days and topics (e.g., `Day01-05_HTML`, `Day26-35_JavaScript`).
   - Each folder contains:
     - **Code Samples**: Example code for the topics covered.
     - **Projects**: Mini-projects or exercises to practice the concepts.
     - **Notes**: Key takeaways or references for quick revision.

4. **Study and Practice**:
   - Follow the daily schedule in the **Learning Plan** section.
   - Use the code in each folder to understand concepts, then try modifying or rewriting it.
   - Complete the projects in the respective folders to apply what you’ve learned.

5. **Use Recommended Resources**:
   - Check the **Recommended Resources** section for tutorials and courses.
   - Supplement your learning with documentation, YouTube tutorials, or communities on platforms like X.

6. **Track Your Progress**:
   - Create your own GitHub repository to store your code and projects.
   - Commit your work daily using Git (e.g., `git add .`, `git commit -m "Day X: Learned Y"`, `git push`).
   - Optionally, fork this repository and contribute your own improvements or projects.

7. **Engage with the Community**:
   - Share your progress or ask questions on platforms like X or developer forums.
   - Check the `Issues` or `Discussions` tab in this repository for community Q&A or updates.

8. **Customize Your Journey**:
   - Feel free to adjust the pace or focus on specific technologies (e.g., choose ReactJS over AngularJS, or MongoDB over MySQL).
   - Use the project ideas in **Days 95-100** to build your portfolio.

## Learning Plan

### Front-End Development (Day 1 - Day 65)

#### HTML (Day 1 - Day 10)
- **Days 1-5: Introduction to HTML**
  - Basics, tags, attributes, graphics.
- **Days 6-10: Advanced HTML / HTML5**
  - APIs, DOM, audio/video, HTML5, MathML.

#### CSS (Day 11 - Day 20)
- **Days 11-15: Introduction to CSS**
  - Basics, properties, selectors, functions.
- **Days 16-20: Advanced CSS**
  - Advanced CSS, SASS, plugins.

#### Bootstrap & Tailwind CSS (Day 21 - Day 23)
- **Day 21: Introduction to Bootstrap & Tailwind CSS**
  - Bootstrap basics, Tailwind CSS (layout, backgrounds, borders, alignment, spacing, sizing, typography).
- **Days 22-23: Advanced Bootstrap & Tailwind CSS**
  - Bootstrap 4/5, Tailwind Flexbox, Grid, effects, filters, tables, transitions, animations, transforms, interactivity, SVG.

#### Front-End Projects (Day 24 - Day 25)
- Build a **Parallax Website** and a **Restaurant Website**.

#### JavaScript (Day 26 - Day 47)
- **Days 26-35: Introduction to JavaScript**
  - Basics, operators, loops, debugging, array/string methods.
- **Days 36-45: Advanced JavaScript**
  - Math, Date, Boolean, DataView, forms, DOM.
- **Days 46-47: JavaScript Projects**
  - **Task Tracker**, **Student Grade Calculator**.

#### Front-End Frameworks (Day 48 - Day 65)
- Choose **ReactJS** or **AngularJS**.
- **ReactJS**: Fundamentals, components, props & states, hooks, apps.
- **AngularJS**: Basics, directives, filters, functions.

---

### Back-End Development (Day 66 - Day 85)

#### Back-End Languages (Day 66 - Day 75)
- Choose one: **Python**, **PHP**, **Node.js**, or **Ruby**.
- **Python**: Input/output, operators, data types, control flow, functions, OOP, exception/file handling.
- **PHP**: Basics, functions, error handling, JSON, image processing, math, MySQLi, strings.
- **Node.js**: Basics, assert, buffer, console, crypto, DNS, file system, HTTP, OS.
- **Ruby**: Basics, OOP, exceptions, control statements, regex, modules, methods, collections.

#### APIs (Day 76 - Day 78)
- Learn APIs, REST APIs, and integration.

#### Back-End Frameworks (Day 79 - Day 85)
- Choose based on language:
  - **Django (Python)**: Basics, views, forms, templates, models.
  - **Laravel (PHP)**: Routing, controllers, Blade, middleware, auth, database, CRUD, APIs.
  - **Express.js (Node.js)**: App creation, express functions, requests, responses, routers.
  - **Ruby on Rails (Ruby)**: Rails, Sass/HAML, jQuery/CoffeeScript, RSpec/TDD.

---

### Databases (Day 86 - Day 95)

#### Version Control (Day 86)
- Learn **Git** and **GitHub**.

#### Databases (Day 87 - Day 95)
- Choose one: **MongoDB**, **MySQL**, or **Firebase**.
- **MongoDB**: Installation, basics, methods, operators, documents/collections, indexing, projects.
- **MySQL**: DDL, DQL, DML, DCL, TCL, joins, WITH clause, ALTER, triggers, GROUP BY, views, ORDER BY, INSERT, sequences.
- **Firebase**: Setup, user sign-in, Cloud Firestore, messages, security rules.
